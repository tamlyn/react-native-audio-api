# ChannelSplitterNode

The `ChannelSplitterNode`, often used in conjunction with its opposite,[`ChannelMergerNode`](/docs/effects/channel-merger-node),
separates the different channels of an audio source into a set of mono outputs. This is useful for accessing each channel separately, e.g., for performing channel mixing where gain must be separately controlled on each channel.

The `numberOfOutputs` is defined by a parameter passed to the [`BaseAudioContext.createChannelSplitter(numberOfOutputs)`](/docs/core/base-audio-context#createchannelsplitter). 
This number in most cases should be equal to the number of channels of the input signal. 
If you pass a mono signal to a six-output splitter then output 0 will hold the mono signal and outputs 1-5 will be silent. 
On the other hand when you pass a 5.1 signal into a two-output splitter, the data from channel 0 and 1 will be copied into outputs 0 and 1 and the rest of the channels will be ignored.
