# ChannelMergerNode

The `ChannelMergerNode`, often used in conjunction with its opposite,[`ChannelSplitterNode`](/docs/effects/channel-splitter-node),
combines multiple mono inputs into a single output channel. This is useful for mixing audio signals from different sources, allowing for more complex audio routing and processing.

The `numberOfInputs` is defined by a parameter passed to the [`BaseAudioContext.createChannelMerger(numberOfInputs)`](/docs/core/base-audio-context#createchannelmerger). 
This number in most cases should be equal to the number of inputs you want to connect to the merger.
If you create a six-input merger and only connect one node to input 0, the output will be a 5.1 signal with channel 0 containing the audio from the connected node and the rest of the channels being silent.
If you connect a non-mono signal to the merger (e.g., a stereo signal to input 0), the signal will be down-mixed into mono before merging.
